# SaaS 客户端界面（灵数 API 开放平台）

## 概述

本系统为 **灵数 API 开放平台**，面向 B 端企业提供大模型 API 接入申请、账号登录及日常管理能力，并为平台运营人员提供租户与申请等管理能力。

| 端 | 说明 | 原型 |
| --- | --- | --- |
| **企业工作台** | 企业用户：模型、密钥、用量等 | `src/prototypes/untitled-3`（侧栏 **功能菜单**） |
| **SaaS 平台管理** | 平台运营：租户、企业申请、字典、公告等 | 同上（侧栏 **平台管理**；原独立 `saas-admin` 已融入） |

企业工作台不含运营管理端能力；平台管理与 **运营管理端** 分工见各节说明。
- **金额展示规则（全局）**：
  - **模型官方价格（Token 单价）**：**/ M Tokens** 保留 **3 位小数**；**/ K Tokens** 保留 **6 位小数**（运营端存储为 1M 维度，切换 K 时按 ÷1000 换算）；按次计费每次价格保留 **3 位小数**；视频输出仍为 **CNY / 1 Second**（展示 3 位小数）
  - **其余 CNY 类金额**（余额、消费、充值、列表费用及汇总等）：**2 位小数**（四舍五入）
  - Token 数量为整数

## 详细内容

### 1. 业务逻辑

#### （1）访问与身份

- 用户首次访问默认进入 **企业登录** 页
- **新企业**（尚未在运营管理端开通账号）：通过登录页入口进入 **企业接入申请**，填写申请信息并提交
- **已开通账号的企业**：在 SaaS 本地已有账号，或首次登录时从运营管理端同步账号后，使用企业账号登录进入主工作台

#### （2）企业接入申请

- 新企业初次访问需填写具体申请信息，点击 **提交申请** 后信息保留
- 提交成功后展示 **申请已提交** 结果页，并提示后续流程：**填写申请 → 运营审核 → 开通账号**
- 申请数据供 **SaaS 管理端** 与 **运营管理端** 使用：
  - **SaaS 管理端**：列表展示企业申请，默认状态 **未处理**；运营人员在运营管理端完成开通后，于管理端点击 **已处理**；未处理申请支持 **删除**（二次确认）
  - **运营管理端**：相关界面 **每 5 秒轮询** 获取申请信息，完成审核与 **项目管理（SaaS 类型）** 账号配置（不在本客户端范围内）
- **账号落库时机**：运营管理端完成申请处理后，企业在运营管理端 **项目管理（类型为 SaaS）** 中配置账号；企业 **首次登录** 时，SaaS 系统再将该账号 **同步写入** 本系统租户管理 / 用户管理（见第 3 节）

#### （3）登录后会话

- 登录成功后默认进入 **模型广场**
- 顶部展示当前 **企业名称**，支持 **退出登录**
- 未登录用户访问需登录的功能页时，自动跳转回登录页

### 3. 企业登录

#### （1）功能描述

企业通过 **账号 / 密码** 登录 SaaS 工作台。点击 **登录** 后，系统按 **先查本地、再查运营管理端、按需同步落库** 的规则处理（见下节）。

- 登录项：**账号**（必填，对应用户名称）、**密码**（必填）
- 操作：**登录**
- 未开通 / 未在运营管理端开通账号的企业：提示 **当前企业未接入申请，请申请！**，并可进入 **企业接入申请**

#### （2）登录处理流程

用户点击 **登录** 后，按以下顺序处理：

| 步骤 | 说明 |
| ---- | ---- |
| 1 | **优先查询 SaaS 系统本地**：当前系统中是否已存在该账号（用户名称） |
| 2a | **本地已存在**：按本地租户 / 用户数据校验密码并登录（常规登录） |
| 2b | **本地不存在**：发起接口请求，查询 **运营管理端 → 项目管理** 中 **项目类型为 SaaS** 的记录，并在其 **账号信息** 中匹配 **用户名称 + 用户密码** |
| 3a | **运营管理端存在匹配账号**：在 SaaS **租户管理**、**用户管理** 中 **同步创建** 对应数据（字段规则见下节），创建成功后完成登录 |
| 3b | **运营管理端不存在匹配账号**：提示 **当前企业未接入申请，请申请！**（不创建本地数据） |

> 前提：企业已在运营管理端完成接入申请审核及账号配置；仅提交申请、尚未在运营管理端开通账号的，登录时走步骤 3b。

#### （3）首次登录同步规则（运营管理端 → SaaS）

当步骤 2b 在运营管理端查到匹配账号时，**同时** 写入 SaaS **租户管理** 与 **用户管理**：

**租户管理（新建一条）**

| 字段 | 取值规则 |
| ---- | -------- |
| 租户名称 | 运营管理端 **企业名称** |
| 租户套餐 | 运营管理端 **套餐类型** 字段 |
| 联系人 | 运营管理端对应联系人 |
| 联系电话 | 运营管理端对应联系电话 |
| 用户名称 | 运营管理端账号信息中的 **用户名称**（与本次登录账号一致） |
| 用户密码 | 运营管理端账号信息中的 **用户密码**（与本次登录密码一致） |
| 账号额度 | 默认 **999** |
| 过期时间 | 默认 **当前登录时间点的一年后** |
| 租户状态 | 默认 **启用** |
| 创建时间 | **当前用户登录时间** |

**用户管理（在该租户下新建一条）**

| 字段 | 取值规则 |
| ---- | -------- |
| 所属租户 | 上一步新建的租户 |
| 用户名称 | 与运营管理端 **用户名称** 一致 |
| 用户昵称 | 与 **用户名称** 保持一致 |
| 用户密码 | 与运营管理端 **用户密码** 一致 |
| 用户状态 | 默认 **开启** |
| 创建时间 | **当前用户登录时间** |

同步完成后，按正常已登录会话进入工作台（默认 **模型广场**）。

#### （4）失败提示

| 场景 | 提示文案 |
| ---- | -------- |
| 本地不存在且运营管理端无匹配账号 | **当前企业未接入申请，请申请！** |

### 4. 企业接入申请

#### （1）功能描述

新企业填写接入申请信息；提交后等待运营管理端审核，并在运营管理端 **项目管理（SaaS 类型）** 中配置企业账号；企业 **首次登录** 时由 SaaS 系统同步创建租户与用户（见第 3 节）。

#### （2）申请表单字段


| 字段       | 必填  | 校验/说明           |
| -------- | --- | --------------- |
| 企业名称     | 是   | 与营业执照一致         |
| 统一社会信用代码 | 否   | 选填；填写时为 18 位信用代码 |
| 联系人      | 是   | 业务对接人           |
| 联系电话     | 是   | 11 位手机号         |
| 联系邮箱     | 是   | 有效邮箱格式，用于接收审核通知 |
| 使用场景说明   | 是   | 多行文本，最多 500 字   |


- 提交前提示：申请信息将保留；运营管理端每 5 秒轮询主动获取；SaaS 管理端支持列表展示与后续处理
- 操作：**返回登录**、**提交申请**

#### （3）提交成功页

- 展示成功状态与说明：申请已保存，运营端轮询获取并处理；审核通过后由运营管理端维护企业账号，企业 **首次登录** 时自动同步至 SaaS 租户与用户
- 提示后续流程：**填写申请 → 运营审核 → 运营开通账号 → 企业登录**（当前处于 **运营审核** 阶段）
- 操作：**返回登录**

### 5. 模型广场

#### （1）功能描述

企业用户以 **卡片网格** 浏览运营管理端维护的已接入模型；**列表不展示价格**，点击卡片打开 **右侧详情抽屉**，查看完整 **基本信息** 与 **官方价格**。

展示字段与运营管理端「模型官方价格维护」的 **基本信息**、**官方价格管理** 保持一致（只读，不含渠道价格）。本客户端不支持模型编辑、订购或渠道价查看。

- 页面说明：浏览已接入模型，按条件筛选后点击卡片查看完整官方价格
- **模型名称** 在列表卡片与详情标题处均支持 **一键复制**
- **厂商品牌标识**：按厂商类型展示对应品牌 Logo（图标资源来自 [Lobe Icons](https://lobehub.com/icons) `@lobehub/icons-static-svg@1.62.0`，原型内打包为本地 SVG；业务映射见下文「厂商标识规则」）

**筛选交互规则**：

- 筛选区 **默认折叠**；点击「筛选」展开/收起
- 展开区顶部提示：各条件支持多选；未选择表示不限制，多条件之间为且关系
- 操作：**查询**（应用当前表单条件、刷新列表、收起筛选区）、**重置**（清空条件并恢复全部模型）、**收起**（仅折叠筛选区，不改变已生效条件）

- 工具栏右侧常显 **匹配模型数量**（如「25 个模型」）
- 无匹配结果时展示空状态「未找到匹配的模型」

#### （3）价格维度（仅详情）

官方价格在 **详情抽屉** 内展示；**列表卡片不展示价格**，亦不提供价格维度切换。


| 维度         | 说明                          |
| ---------- | --------------------------- |
| / M Tokens | 默认维度；与运营管理端存储的官方 Token 单价一致 |
| / K Tokens | 在 1M 维度基础上 **÷1000** 换算后展示  |


- 单价展示精度：**/ M Tokens** 保留 **3 位小数**；**/ K Tokens** 保留 **6 位小数**（见概述「金额展示规则」）
- 仅 **按 Token** 计费的模型在详情内展示维度切换（Segmented：**/ M Tokens** / **/ K Tokens**）
- **按次数** 计费的模型不参与 Token 维度切换；官方价格区标题固定为「官方价格（CNY）」

每条模型以可点击卡片展示；点击卡片或按 Enter / Space 打开详情抽屉。卡片内 **模型名称** 的复制操作不触发打开详情。


| 信息项     | 说明                                                            |
| ------- | ------------------------------------------------------------- |
| 厂商 Logo | 按厂商类型展示品牌标识                                                   |
| 模型名称    | 支持复制；过长省略                                                     |
| 厂商类型    | 字典标签                                                          |
| 模型类型    | 向量模型 / 文本模型 / 图像生成 / 视频生成 / 音频模型                              |
| 计费模式    | 按 Token / 按次数                                                 |
| 模型介绍    | 对应运营端「模型官方价格管理」中的 **备注**；有内容时最多展示 2 行（超出省略）；无内容时保留占位以保持卡片高度一致 |
| 更新时间    | 官方价格维护时间，精确到 **日期**（YYYY-MM-DD）                               |
| 查看详情    | 卡片底部操作提示，点击进入详情                                               |


列表 **不展示** 官方价格摘要、输入/输出单价或每次价格。

#### （5）模型详情（右侧抽屉）

从列表点击卡片后，以 **右侧抽屉** 打开详情（桌面端宽度约 720px；小屏全宽）。标题区：**{模型名称}** + 「模型详情」；模型名称支持复制。

分为 **基本信息**、**官方价格** 两个区块。

| 属性网格         | 见下表                                  |


属性网格字段（**不含**模型名称，名称仅在抽屉标题展示）：


| 字段   | 说明                                     |

| 更新日期 | 取自运营管理端「模型价格管理」**更新时间**，截取到 YYYY-MM-DD |


**官方价格**

- 区标题：按 Token 时为「官方价格（/ M Tokens）」或「官方价格（/ K Tokens）」；按次数时为「官方价格（CNY）」
- **非阶梯价格**：顶部 **价格摘要条**（按 Token 展示输入/输出两项；按次数展示每次价格）+ 下方 **价格明细卡片网格**（仅展示已维护的非空字段）
- **阶梯价格**（`tierPricing = true` 且存在输入区间配置）：按 **输入区间** 分段展示；每段包含区间标签、该段价格摘要条及明细网格

输入区间展示规则（与运营端一致）：**(下限, 上限]**，单位为 k Tokens；未填上限时为 **(下限, +∞)**。

**按 Token — 可展示字段**（非空才展示；数值随详情内价格维度切换）：


| 字段         | 单位说明                                |
| ---------- | ----------------------------------- |
| 输入价格       | / M Tokens 或 / K Tokens             |
| 输出价格       | 同上                                  |
| 文本缓存写入价格   | 同上                                  |
| 文本缓存输入命中价格 | 同上                                  |
| 图像输入价格     | 同上                                  |
| 图像输出价格     | 同上                                  |

### 7. 企业密钥

#### （1）功能描述

管理企业 API 密钥；企业侧支持 **启用 / 禁用**（仅控制企业工作台侧是否允许发起调用，不回写运营管理端）。

密钥能否 **实际调用成功**，须同时满足 **「密钥调用成功前提」**（见下节）；企业侧禁用只是前提之一。

密钥列表需通过运营管理端 **增量获取**。页面查询区上方提供 **接口调用获取密钥** 按钮：点击后向运营管理端发起请求，**增量拉取**当前登录企业对应项目、且企业工作台尚未持有的密钥数据。**仅增量写入**的密钥，在企业侧 **状态默认为已启用**；已存在的密钥不覆盖、不重置状态。

#### （2）密钥调用成功前提

企业使用密钥发起 API 调用，须 **同时满足** 以下全部条件，方可调用成功：


| #   | 条件             | 说明                                        |
| --- | -------------- | ----------------------------------------- |
| 1   | 企业侧状态为 **已启用** | 企业工作台操作列 **启用 / 禁用** 控制                   |
| 2   | 管理端状态为 **已启用** | 运营管理端密钥配置状态；**已禁用** 不可调用                  |
| 3   | 额度可用           | **有限额度**：剩余额度 &gt; 0（未耗尽）；**无限额度**：不受额度限制 |
| 4   | 分组匹配           | 密钥所属分组与路由到的 **渠道分组一致**                    |
| 5   | 分组链路启用         | 上述 **密钥分组**、**渠道分组** 均为 **启用** 状态         |


任一条件不满足时，调用失败；由网关/服务端按上述规则校验。

> 管理端「已耗尽」视为管理端状态不可用（通常对应有限额度剩余 ≤ 0）。

#### （3）增量获取密钥


| 项      | 说明                                                                                                                               |
| ------ | -------------------------------------------------------------------------------------------------------------------------------- |
| 入口     | 查询区上方 **接口调用获取密钥** 按钮                                                                                                            |

| 请求     | 调用运营管理端接口，返回当前企业在管理端的 **完整系统密钥列表**（用于与企业工作台比对）                                                                                   |
| 比对规则   | **系统密钥**：企业端有、管理端无 → 标记删除；管理端有、企业端无 → 增量追加；**自建密钥** 不参与比对与删除                                                                     |
| 删除确认   | 存在待删除系统密钥时，弹窗提示：「当前密钥名称为 {名称} 系统管理端已删除，此次操作将删除此密钥，请确认是否继续。」（多条时列表展示）；**是** → 删除对应密钥并执行增量追加；**否** → 取消本次同步，不做任何变更                 |

### 8. 用量查询

#### （1）功能描述

查询企业 API **消耗** 调用记录，展示账户余额与累计消费；支持条件筛选与自动刷新。

#### （2）系统密钥账户概览

与系统密钥相关调用日志及账户余额同步，**默认每 5 秒请求接口** 刷新系统密钥数据（自建密钥消耗记录不随轮询更新），展示最近数据更新时间：


| 指标   | 说明                  |
| ---- | ------------------- |
| 当前余额 | 企业账户剩余额度（CNY，2 位小数） |
| 累计消费 | 企业累计消费总额（CNY，2 位小数） |


#### （3）查询条件

| 状态 | **未处理** / **已处理** |
| 操作 | 见下节 |


#### （4）操作规则


| 操作 | 条件 | 说明 |
